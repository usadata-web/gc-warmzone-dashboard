Methodology
Data Processing & Classification

Source: USAspending full database via Microsoft Fabric (Delta Lake). 75 NAICS codes, FY2022-2026, AWARD records only (IDV ceiling values excluded).

Step 1 deduplication: One row per award_id_piid (removes duplicate transaction mods). GC-addressable CAGR: contracts <$250M only — the market GC can compete in. April 2026 data refresh.

Three-layer classification:

  • Layer 1 — NAICS code: Coarse sector assignment
  • Layer 2 — PSC code: Structured middle filter
  • Layer 3 — Award description keywords: Precision classification (16 categories, ~30 sub-categories, ~460 stopwords)

Track A (61 narrow NAICS): Classified by NAICS code alone — code is specific enough to determine sector.

Track B (14 broad NAICS): Classified by keyword regex patterns — 63.3% classified after 5 iterations on deduped data. 36.7% unclassified (military codenames, empty descriptions, opaque text).

Equal-Weight Principle

Federal spending data is used for context and validation — to confirm real demand and identify geographic concentrations. It is NOT used to rank sector priority. All 8 GC capability categories start on equal footing. A $5B sector with exceptional growth may rank higher than a $50B sector with flat trends.

"Warm Not Hot" Standard

Targets must be in sectors with established, durable demand — not hype cycles, policy spikes, or unscaled emerging tech. Yes: PFAS remediation, IT managed services, facilities maintenance. No: AI startups, drone companies, pandemic-driven health spending.
